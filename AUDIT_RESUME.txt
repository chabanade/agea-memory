================================================================================
                          TOP 5 RISQUES CRITIQUES
================================================================================

🔴 RISQUE #1 : Perte Neo4j (SPOF Critique)
   Impact      : Perte mémoire permanente (entités, relations, faits)
   Probabilité : Moyenne (corruption index, OOM possible)
   Durée       : Irrécupérable
   Fix         : Backup quotidien Neo4j (1-2h)
   Criticité   : TRÈS CRITIQUE

🔴 RISQUE #2 : Perte PostgreSQL (SPOF Critique)
   Impact      : Perte conversations + queue Graphiti
   Probabilité : Moyenne (disque plein, corruption)
   Durée       : 2+ heures (depends on S3 backup)
   Fix         : Réplication Primary/Standby (4-8h)
   Criticité   : CRITIQUE

🔴 RISQUE #3 : Zéro Monitoring (Invisible Failures)
   Impact      : Mehdi découvre bugs via plaintes utilisateurs
   Probabilité : Certaine (actuellement aveugle)
   Durée       : Indéfinie sans alertes
   Fix         : Prometheus + Grafana (4h)
   Criticité   : CRITIQUE

🔴 RISQUE #4 : Timeout 60+ secondes possible (LLM)
   Impact      : Utilisateur voit "bot typing" pendant 1+ min
   Probabilité : Moyenne (si DeepSeek + Qwen down)
   Durée       : 60-90 secondes par message
   Fix         : Circuit breaker (2h)
   Criticité   : HAUTE

🔴 RISQUE #5 : Gemini Quota Exhausted
   Impact      : Service dégradé (pas d'embeddings) si surge
   Probabilité : Haute (1000/jour = limité)
   Durée       : 24 heures (quota reset) ou fallback Ollama
   Fix         : Cache embeddings + Ollama fallback (3h)
   Criticité   : HAUTE

                        ANSWERS AUX QUESTIONS CLC
================================================================================

Q1: "Combien de SPOF?"
A: 6 points de défaillance uniques
   • PostgreSQL local volume
   • Neo4j local volume
   • Gemini API quota
   • Caddy single instance
   • VPS Hostinger single instance
   • Telegram API (hors contrôle)

Q2: "Scalabilité?"
A: Max 3-5 utilisateurs concurrents avant bottleneck
   Bottleneck = asyncpg pool (max_size=3 connexions)
   Pour 10 users : pool=20, Neo4j=2G, PG replication
   Pour 50+ users : Kubernetes + sharding

Q3: "Monitoring?"
A: ❌ Zéro monitoring actuellement
   Ne pas visible : latence LLM, queue backlog, quota utilisé, errors
   Mehdi découvre bugs via plaintes utilisateurs

Q4: "Coût fix?"
A: Semaine 1 = €600-1000 (1 dev)
   Mois 1 = €3000-4000 (1 DevOps + 1 dev)
   Production (10+ users) = €6000-8000 + infrastructure

Q5: "Quand est risqué?"
A: MAINTENANT si:
   • > 5 utilisateurs concurrents attendus
   • Données Neo4j irremplaçables
   • SLA > 99% attendu
   • Perte données acceptable = non
